📊 Financial Data Pipeline – Medallion Architecture with Spark + dbt
Overview

This project implements an end-to-end Data Engineering pipeline for processing brokerage notes and transforming raw financial data into a structured analytical model.

The solution follows the Medallion Architecture (Bronze → Silver → Gold) and combines Apache Spark for data processing with dbt for dimensional modeling, resulting in a modern analytics engineering workflow.

The final output is a Star Schema optimized for analytical queries and BI usage, built on top of locally stored Parquet datasets and a DuckDB analytical layer.

All sensitive information (such as account identifiers and source filenames) has been anonymized for portfolio purposes.

🚀 Project Goals
Parse brokerage notes generated by Brazilian brokers.
Build a reproducible ETL pipeline using Python and Spark.
Implement the Medallion Architecture (Bronze, Silver, Gold).

Demonstrate modern Data Engineering practices (Spark + dbt).

🥉 Bronze Layer

The Bronze layer stores raw extracted brokerage data with minimal transformation.

Responsibilities:

Raw ingestion from brokerage files
Data traceability
Preservation of original structure
🥈 Silver Layer (Spark Processing)

The Silver layer is built using Apache Spark, responsible for:

Parsing and structuring raw financial data
Standardizing assets and operations
Normalizing numeric values (fees, quantities, prices)
Allocating brokerage fees
Writing optimized Parquet datasets
Key Output
Clean, structured transactions dataset
Column-level normalization applied
Ready for analytical modeling
🥇 Gold Layer (dbt Dimensional Model)

The Gold layer is built using dbt, implementing a Star Schema model.
