# 📊 Financial Data Pipeline — Modern Data Engineering with Spark & dbt

## Overview

This project implements an end-to-end Data Engineering pipeline for processing Brazilian brokerage notes into an analytical data model.

Instead of building a traditional ETL using only Python, the project adopts a modern data stack inspired by production environments:

* Python for ingestion
* Apache Spark for distributed data processing
* Medallion Architecture (Bronze → Silver → Gold)
* dbt for dimensional modeling
* DuckDB as a lightweight analytical warehouse
* Parquet as the storage format

The final result is a complete analytical pipeline capable of transforming raw brokerage statements into a clean Star Schema ready for BI, analytics and future machine learning workloads.

# 🥉 Bronze Layer

Raw ingestion layer.

Responsibilities:

* Preserve original data
* Maintain traceability
* Store brokerage notes without business transformations

---

# 🥈 Silver Layer (Apache Spark)

The Silver layer is entirely processed using Apache Spark.

Main transformations include:

* Data cleansing
* Type standardization
* Asset normalization
* Brokerage fee allocation
* Financial calculations
* Source file anonymization
* Curated Parquet generation

Spark was chosen to simulate a production-grade distributed ETL engine rather than relying exclusively on local Python processing.

---

# 🥇 Gold Layer (dbt)

The Gold layer is modeled using dbt following Kimball dimensional modeling principles.

## Dimensions

* dim_date
* dim_asset
* dim_operation
* dim_brokerage

Each dimension uses surrogate keys generated by dbt.

---

## Fact Table

### fact_trades

The fact table stores every financial transaction.

Features:

* Incremental loading
* Duplicate prevention
* Foreign keys to all dimensions
* Optimized analytical structure

Measures include:

* Quantity
* Unit Price
* Gross Value
* Settlement Fee
* Brokerage Fee
* Emoluments
* Asset Transfer Fee
* Total Fees
* Total Cost
