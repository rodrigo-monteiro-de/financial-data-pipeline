# рҹ“Ҡ Financial Data Pipeline вҖ“ Medallion Architecture with Python

## Overview

This project implements an end-to-end **Data Engineering pipeline** for processing brokerage notes and transforming raw financial data into a structured analytical model.

The solution follows the **Medallion Architecture (Bronze вҶ’ Silver вҶ’ Gold)** and applies **dimensional modeling (Star Schema)** to prepare investment transactions for analytics and future AI/LLM use cases.

> **Note:** All sensitive information (such as account identifiers and source filenames) has been anonymized for portfolio purposes.

---

# рҹҡҖ Project Goals

* Parse brokerage notes generated by Brazilian brokers.
* Build a reproducible ETL pipeline using Python.
* Apply the Medallion Architecture.
* Create a clean dimensional model for analytics.
* Persist curated datasets in Parquet format.
* Demonstrate Data Engineering best practices and modular project organization.

# рҹҘү Bronze Layer

The Bronze layer stores raw extracted data from brokerage notes with minimal transformation.

Typical responsibilities:

* Raw ingestion
* Traceability
* Original data preservation

---

# рҹҘҲ Silver Layer

The Silver layer applies business rules and data quality improvements, including:

* Data cleansing
* Standardized asset names
* Fee allocation
* Numeric normalization
* Transaction consolidation
* Structured Parquet output

---

# рҹҘҮ Gold Layer

The Gold layer implements a **dimensional model (Star Schema)** optimized for analytical workloads.

## Dimensions

### `dim_brokerage`

| brokerage_key | brokerage |
| ------------- | --------- |
| 1             | RICO      |

### `dim_asset`

Contains unique traded assets and their surrogate keys.

### `dim_operation`

| operation_key | operation | description |
| ------------- | --------- | ----------- |
| 1             | B         | Buy         |
| 2             | S         | Sell        |

### `dim_date`

Contains:

* `date_key`
* `trading_date`
* `year`
* `month`
* `day`
* `quarter`

---

## Fact Table

### `fact_trades`

Stores investment transactions linked to dimensions through surrogate keys.

Key fields include:

* `date_key`
* `brokerage_key`
* `asset_key`
* `operation_key`
* `quantity`
* `unit_price`
* `gross_value`
* `settlement_fee`
* `emoluments`
* `brokerage_fee`
* `asset_transfer_fee`
* `total_fees`
* `total_cost`
* `source_file` *(anonymized for traceability)*
